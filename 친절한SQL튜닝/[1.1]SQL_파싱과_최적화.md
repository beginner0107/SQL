# SQL 파싱과 최적화

## 1.1.1 구조적, 집합적, 선언적 질의 언어

- SQL은 "Structured Query Language"의 줄임말
- 구조적 질의 언어
- 원하는 결과집합을 ```구조적, 집합적```으로 선언하지만, 그 결과집합을 만드는 과정은 ```절차적```
- ```프로시저```가 필요
  - 그런 프로시저를 만들어 내는 DBMS 내부 엔진이 ```SQL 옵티마이저```
    1. 사용자가 SQL을 작성한다.
    2. 옵티마이저가 DBMS 내부에서 프로시저를 작성하고 컴파일해서 실행 가능한 상태로 만든다.
       - 이 과정을 ```SQL최적화```라고 한다.
